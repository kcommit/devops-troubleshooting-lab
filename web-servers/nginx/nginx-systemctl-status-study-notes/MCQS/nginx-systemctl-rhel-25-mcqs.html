Nginx MCQ Quiz

Q1. What is the correct command to check the Nginx service status using systemd?

Correct answer: sudo systemctl status nginx
Explanation: The correct format is systemctl <action> <service-name>, so the command is sudo systemctl status nginx.

Q2. In systemctl status nginx, what does Active: active (running) mean?

Correct answer: Nginx is currently running
Explanation: active (running) means the Nginx service is working right now.

Q3. What does Loaded: loaded mean in the Nginx service status output?

Correct answer: systemd found the Nginx service file
Explanation: Loaded: loaded means systemd found the service unit file for Nginx.

Q4. What does enabled mean in the Loaded line?

Correct answer: Nginx will start automatically at boot
Explanation: enabled means the service is configured to start automatically when the system boots.

Q5. Which line is usually the most important when checking if Nginx is running?

Correct answer: Active: active (running)
Explanation: The Active line tells whether the service is running, stopped, or failed.

Q6. What does status=0/SUCCESS usually mean?

Correct answer: The command completed successfully
Explanation: status=0/SUCCESS means the command finished without an error.

Q7. What does ExecStartPre=/usr/sbin/nginx -t show?

Correct answer: Nginx tested its configuration before starting
Explanation: ExecStartPre runs before starting the service; for Nginx it commonly tests the configuration.

Q8. Which command manually tests Nginx configuration?

Correct answer: sudo nginx -t
Explanation: sudo nginx -t checks whether the Nginx configuration syntax is valid.

Q9. What does Main PID: 209 (nginx) mean?

Correct answer: 209 is the main Nginx process ID
Explanation: PID means Process ID. The Main PID is the main process managed by systemd.

Q10. In Nginx, what is the job of worker processes?

Correct answer: They handle user/web requests
Explanation: Worker processes handle website traffic and client requests.

Q11. What is the role of the Nginx master process?

Correct answer: It controls Nginx worker processes
Explanation: The master process controls and manages Nginx worker processes.

Q12. What does the Memory: 10.5M line show?

Correct answer: Nginx memory usage
Explanation: The Memory line shows how much memory the service is currently using.

Q13. Which command can test Nginx from the terminal after it starts?

Correct answer: curl localhost
Explanation: curl localhost sends a request to the local web server.

Q14. What URL can you open in a browser to test local Nginx?

Correct answer: http://localhost
Explanation: Nginx serving locally can be tested by opening http://localhost.

Q15. On Ubuntu/Debian, what user does Nginx commonly run worker processes as?

Correct answer: www-data
Explanation: Ubuntu/Debian commonly use the www-data user and group for Nginx.

Q16. On RHEL/Rocky/AlmaLinux/CentOS/Fedora, what user does Nginx commonly use?

Correct answer: nginx
Explanation: RHEL-based systems commonly use the nginx user and group.

Q17. Which command checks the Nginx user entry on RHEL?

Correct answer: grep nginx /etc/passwd
Explanation: On RHEL, search for nginx in /etc/passwd.

Q18. Which command checks the Nginx group entry on RHEL?

Correct answer: grep nginx /etc/group
Explanation: /etc/group stores Linux group information.

Q19. Why do we use sudo with grep nginx /etc/shadow?

Correct answer: /etc/shadow requires root permission
Explanation: /etc/shadow contains password information and is protected, so normal users cannot read it.

Q20. In /etc/shadow, what does !! usually mean for the nginx account?

Correct answer: Password is locked
Explanation: !! means the account password is locked, which is normal for a service account.

Q21. What does /sbin/nologin mean in /etc/passwd?

Correct answer: The account cannot be used for normal shell login
Explanation: /sbin/nologin prevents normal shell login for service accounts.

Q22. Which file usually contains the Nginx user directive?

Correct answer: /etc/nginx/nginx.conf
Explanation: The main Nginx configuration file is usually /etc/nginx/nginx.conf.

Q23. On RHEL, what output might grep user /etc/nginx/nginx.conf show?

Correct answer: user nginx;
Explanation: On RHEL-based systems, the config commonly shows user nginx;

Q24. Why does the Nginx master process often start as root?

Correct answer: To bind to privileged ports like 80 and 443
Explanation: Ports below 1024, such as 80 and 443, require privileged permissions.

Q25. Which command shows actual running Nginx master and worker processes?

Correct answer: ps aux | grep nginx
Explanation: ps aux | grep nginx shows running Nginx processes and which user owns them.
